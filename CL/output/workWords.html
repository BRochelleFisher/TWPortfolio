First Words

With three small samples of cybersecurity documentation from different organizations, the Python script gave us 4755 unique words. Your sample should be much larger. Our goal is to create a cybersecurity dictionary of approximately 1,000 words. This does not include product names or company-specific words.

Effort: The average effort is 20 words each hour. We can expect this dictionary to require 238 hours (about 30 working days) to complete.

Procedure 1. To work through the words to create your dictionary:

  1. Import the dictionary CSV file to a spreadsheet.

    We will use Google Sheets. You can use Microsoft Excel or similar.

    Our Python script uses a pipe ( | ) as a delimiter between the word and its frequency of use. When you import the CSV, set the pipe as the delimiter, to get the word list in column A and the frequency in column B.

  2. Change row 1 to be these headers: Word, Count, Part of Speech, Definition, Good Example, Bad Example, Allowed?, Audience, Alt1, Alt2
  3. If there are rows between the header row and the first word, delete them.
  4. The script sorted the list by highest frequency to lowest. Start with the words most used. These will be the easiest.
  5. Set the Part of Speech (PoS).

    • A typical result for the most used words are the name of your organization or product. Set the PoS as Name. This lets you filter for these nouns which may change more often than regular nouns. And, you will have variables for these names, which should be ignored by the checker tool.

    • Enter the PoS for the other most common words. When you get to a word that could be used in multiple ways, enter TBD and stop.

  6. If the word is a name or common word, such as the that this from, you can leave the Definition blank. For all other words:

    • If you do not allow the word, do not enter a definition. Enter an alternative word. If there are multiple alteratives, enter one word in each ALT column. You can add more ALT columns if required. In the Allowed column, enter F (for false).

    • If you allow the word, enter a definition. Leave the ALT columns blank.

  7. If you can easily copy an exmaple of the word, do it now. If the word is not allowed, paste the example from your sample in Bad Example. In Good Example, enter the rewritten phrase or sentence with the correct allowed alternative. If the word is allowed, paste your sample in Good Example, and leave Bad Example blank. In the Allowed column, enter T (for true).
  8. In the Allowed column, enter T (for true) or F (for false).
  9. In the Audience column, enter all, or create a list of your audience personas on a separate tab and then create a data validation for this column, to select the persona from the list.

Examples of the First Words

  • In our example, one of the product or company names is the word with the highest frequency. We set its PoS as name and set it to be Allowed. We set the audience to all. We do not set a value for the other columns yet.

  • In the next top 20 words, we have many that are clearly to be allowed: this that will with from your. These are pronouns, prepositions, and modular verbs. In this case, it does not give a lot of information to define the PoS. Is that a pronoun, adverb, conjunction, or determiner? In a technical writing dictionary for native speakers, it does not matter. We set up a rule that we do not use minimalist rules. If that helps make a sentence easier to understand, we use it. You can set a definition forf will, to make it clear that the use of will as a part of a future-tense verbal phrase is allowed, but it is not allowed as a verb to mean "to desire" or as a noun to mean "legal declaration of disposal of property after death".

  • Next we have words that are industry standard for software technical writing: introduction user data attributes attribute example organisation. At some point, we may require a specific deinition for some of these words. For now, we set the PoS, set to Allowed, and set audience to all.

    Note

    Note that attribute and attributes are two items. We restrict the values of the Word and ALT columns to one word or phrase to prepare for a checker tool that might be simple Python script. If you know that you will use an AI-driven tool, such as writer.com or jasper.ai, change the dictionary to work with the aquired tool.

    See organisation. We know we need this word, but this spelling is British. We know our rules will tell us to use American spelling. In this case, we will make a decision for our dictionary without analysis, or to put it more accurately, despite analysis. The spelling of organisation is the most commonly used form, but we will not use it. We will use the American spelling.

    1. Sort the complete range of dictionary alphabetically, by Word.

    2. Set organization and organizations as allowed (T in Allowed).

    3. Set the British organisation and organisations as not allowed (F in Allowed).

    4. In the allowed words, set the definition: nonspecific body of people with a purpose

    5. In ALT1 for organisation and organisations, enter organization and organizations.

    Note

    Why not add organizational on the fly? Answer: It is used only one time. We will analyze the text and hope to find a more common rewrite in that one sentence.

Sort the range by Count. We are done with the easiest words. Next: analyze words in their syntax and by content to make decisions for the dictionary.
